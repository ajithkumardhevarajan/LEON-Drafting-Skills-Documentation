🎯 Generic AI Assistant Skill
Primary Function: Answer questions, summarize content, extract information
Model: Gemini 2.5 Flash
Input: Story text, documents, or questions
Output: Summaries, quotes, answers, or insights
🔀 Orchestration Layer
The orchestration layer is the intelligent routing system that processes all user requests:

1. Request Reception
User input received through LEON AI Assistant interface

2. Intent Interpretation
• Orchestrator uses Intent Interpreter Prompt to analyze user request
• Determines user intent and required action type
• Classifies request as: Question, Summarization, Extraction, or Skill-specific task

3. Routing Decision
Orchestrator decides the best path:
• Generic Query → Routes to Generic AI Assistant (Gemini 2.5 Flash)
• Urgent Creation → Routes to Urgent skill (GPT-4.1)
• Story Update → Routes to Update skill (Gemini 2.5 Pro)
• New Story → Routes to Blank Slate skill (Gemini 2.5 Pro)

4. Tool Selection
Orchestrator determines if additional tools are needed:
• Archive Search - Semantic search on Reuters Text Archive
• Live Market Data - Real-time market information (backlog)
• Document Access - User-uploaded PDFs or plain text
• Web Content - External sources (backlog)

5. Context Management
• Maintains conversation history
• Tracks user preferences and custom instructions
• Preserves context across multiple interactions
• Manages session state and selected sources
Complete Workflow
Step 1: User Input
User asks question, requests summary, or provides content for analysis

Step 2: Orchestration Processing
• Orchestrator receives request via LEON interface
• Executes Intent Interpreter Prompt to classify intent
• Determines this is a generic query (not skill-specific)
• Selects Gemini 2.5 Flash for fast response
• Checks if archive search or other tools are needed

Step 3: Model Execution
• System calls Gemini 2.5 Flash model (sub-second response)
• Model processes request using appropriate analysis technique:
  - Summarization: Extracts key points and themes
  - Quote Extraction: Identifies and attributes direct quotes
  - Q&A: Searches content for specific information
  - Analysis: Identifies patterns, themes, or insights

Step 4: Response
• Orchestrator receives model output
• Formats response for user presentation
• AI provides answer, summary, or extracted information to user

Step 5: Follow-up (Optional)
• User can ask follow-up questions
• Orchestrator maintains conversation context
• Chat-based interaction for refinement
• Quick iterations without regenerating full drafts
🔀 Orchestration Routing Examples
Example 1: "Summarize this earnings report"
→ Orchestrator classifies as Summarization
→ Routes to Generic AI Assistant (Gemini 2.5 Flash)
→ Returns summary in seconds

Example 2: "Create an urgent from these alerts"
→ Orchestrator classifies as Urgent Creation

Example 3: "Update this story with new revenue figures"
→ Orchestrator classifies as Story Update

Example 4: "Write a story about Tesla earnings"
→ Orchestrator classifies as New Story Creation
→ Routes to Blank Slate skill (Gemini 2.5 Pro)
→ Triggers archive search → Source selection → Full story generation

⚡ Urgent Skill
Primary Function: Build 1-2 sentence urgent news stories from news flashes
Model: GPT-4.1
Input: News flashes (breaking news alerts in decreasing order of importance)
Output: 1-2 sentence urgent (max 80 words) + headline
Complete Workflow
Step 1: User Trigger
User selects news alerts/flashes (ordered by importance)

Step 2: Backend Processing
• System calls GPT-4.1 model
• Executes Body Generation Prompt (400+ lines of Reuters style rules)
• Model processes news flashes and generates 1-2 sentence urgent
• Based on the body, headline is generated
• Model creates headline (sentence case, under 65 characters)

Step 3: Review
Urgent + Headline presented to user for review

Step 4: Refinement (Optional)
• User can unselect or reorder alerts to regenerate
• User can request refinements (calls Refinement Prompt)
• System makes surgical edits based on feedback

Step 5: Insert
User clicks "Insert Draft" button → Content inserted into LEON editor
✓ Key Rules
  • First news flash MUST be in first sentence
  • Use ONLY "said", "added", "showed", "according to"
  • Never assume names, titles, or gender
  • Use ##INSERT NAME## placeholder if unsure
  • Include <DOW_placeholder> for day of week
  • Maximum 80 words, 2 sentences
📋 Style Guidelines
  • Never use 'percent' - always use %
  • Spell out months (August 25, not Aug. 25)
  • $ before figures for USD only
  • Spell out other currencies (500 euros)
  • Never start sentences with numbers
  • Formal tone, no speculation
🤖 Backend Prompts
  • Body Prompt - 400+ lines of Reuters rules
  • Headline generation
  • Refinement Prompt - Type A (explicit) & Type B (requests)
  • All prompts enforce strict Reuters style
🔧 Refinement Options
Type A - Explicit Instructions: User provides exact content
Example: "Change headline to China sanctions European automakers"

Type B - General Requests: User asks for changes without content
Example: "Add CEO's name" - only fulfilled if info exists in news flashes
Example
News Flash: "COLOMBIA'S ECOPETROL IN TALKS WITH OCCIDENTAL TO BUY STAKE IN CROWNROCK"

Urgent Output: "Colombia's Ecopetrol is in talks with Occidental to buy a stake in CrownRock on <DOW_placeholder>."

Headline: "Colombia's Ecopetrol in talks with Occidental on CrownRock stake"
🔄 Update Story Skill
Purpose: Update existing Reuters stories with new information
Model: Gemini 2.5 Pro
Length Target: ~100 words new content (general guideline, not rigid)
Output: Updated story + 25-word advisory explaining changes
Entry Points
  • Click "Update" from story read-only screen
  • Select "Update" from dropdown in search page
  • Right-click "Update" from monitoring page
  • Open story and trigger AI Assistant
  • Select "Publish & Update" when publishing
Complete Workflow
Step 1: User Trigger
User selects existing story and clicks "Update" from multiple entry points

Step 2: User Input Request
• Prompt: "How would you like to proceed with the update? Feel free to add new info, rewrite the story or provide custom instructions..."
• User provides new information/content in input field
• User can upload documents (PDF, plain text)

Step 3: Archive Search (Automatic)
• Based on original story + user input text, system performs semantic search on Reuters Text Archive
• Relevant archive stories automatically displayed to user
• User can select archive stories for additional context
• Options: Load more / Skip / Use Selected and Generate / Use Selected and Provide Info

Step 4: Mode Selection
System asks: Should selected stories and content be added as background or rewrite?
User chooses update mode:
• Add Background - Keep existing lede, add new info to body
• Rewrite - New lede, restructure with new info as primary driver

Step 5: Backend Processing
• System calls Gemini 2.5 Pro model
• Executes Story Update Body Prompt with selected mode
• Model integrates new information with original story following Reuters style
• Generates 25-word Advisory listing paragraph changes
• Adds References linking paragraphs to source inputs

Step 6: Review
AI-drafted update + Advisory + References presented to user

Step 7: Refinement (Optional)
• User can chat with AI for refinements
• System calls Refinement Prompt for surgical edits
• Story Diff Comparison available (in progress)

Step 8: Insert
User clicks "Insert Draft" → Updated story inserted into LEON editor
🔧 Add Background Mode
  • Preserves existing lede (top paragraphs)
  • Seamlessly integrates new info into body
  • Surgically updates if new info overlaps top
  • Avoids repeating facts from paragraphs 1-2
✨ Story Rewrite Mode
  • Treats new info as primary driver
  • Writes new lede with fresh angle
  • Preserves unique reporting from original
  • Moves older material down, not deleted
Key Guidelines
  • Lede must be under 40 words
  • Length guided by newsworthiness, not rigid word count
  • Advisory max 25 words, lists paragraph numbers
  • Maintains Reuters style throughout
  • Can upload documents (PDF, plain text)
  • Story Diff Comparison available (in progress)
🤖 Backend Prompts
  • Story Update Body Prompt - with mode selection
  • Refinement Prompt - surgical edits only
  • Advisory generation - 25-word summary
  • References generation - source citations
  • All prompts preserve journalistic integrity
✨ Blank Slate Skill (Spot Story)
Primary Function: Create complete news stories (~500 words) from provided sources
Model: Gemini 2.5 Pro
Input: New Content Sources + Background Sources from archive
Output: Headline + Bullet points + Complete structured story
Complete Workflow
Step 1: User Trigger
User initiates Blank Slate skill with topic or context

Step 2: Archive Search (Automatic)
• System performs semantic search on Reuters Text Archive
• Returns relevant stories categorized as:
  - New Content Sources (for lead & main story)
  - Background Sources (for context & history)
• User selects which sources to include (can select 15+ stories)

Step 3: User Input
• User can provide additional context or instructions
• User can upload documents (PDF, plain text) for additional info

Step 4: Backend Processing - Article Planning
• System calls Gemini 2.5 Pro model
• Model enters planning phase (wrapped in <article_planning> tags):
  - Identifies primary story angle from New Content Sources
  - Lists key facts driving lead and narrative
  - Notes relevant background context
  - Considers chronology using publication dates
  - Brainstorms 2-3 potential headlines
  - Outlines structure with word counts per section

Step 5: Story Generation
• Executes Spot Story Body Prompt (500-word structured article)
• Generates story following structure:
  - Lead (35 words max)
  - Details (200-250 words)
  - Nut Graph (50-70 words)
  - Quotes (100-150 words, if available)
  - Background (100-150 words)

Step 6: Headline & Bullets Generation
• Based on the story, headline is generated (sentence case, 5-10 words)
• Based on the story, bullet points are generated (3 key points, max 15 words each)
• References are added with source citations per paragraph

Step 7: Review
Complete story + Headline + Bullets + References presented to user

Step 8: Refinement (Optional)
• User can request specific changes
• System calls Refinement Prompt for surgical edits
• User can regenerate with different sources

Step 9: Insert
User clicks "Insert Draft" → Complete story inserted into LEON editor
Story Structure Output (~500 words)
Lead (35 words max): Most newsworthy development from New Content Sources
Details (200-250 words): Expand on lead, provide context across multiple paragraphs
Nut Graph (50-70 words): Why readers should care - significance, what's at stake
Quotes (100-150 words): Direct quotes from sources (only if available - never fabricated)
Background (100-150 words): Historical context from Background Sources, chronologically organized
📝 Source Prioritization
  • New Content Sources → Lead & main story
  • Background Sources → Context & history
  • Prioritize by recency & newsworthiness
  • Use publication dates for timeline
  • Synthesize, don't just summarize
🤖 Backend Prompts
  • Body Prompt - 500-word structured article
  • Headline generation
  • Bullet points generation
  • References generation
  • Refinement Prompt - Surgical edits
Additional Outputs
Headline: 5-10 words, sentence case, under 65 characters
Bullet Points: 3 key points (max 15 words each, no periods)
References: Paragraph-level citation to source inputs

Example bullets: "Probe partly relates to posts following Hamas attacks" / "Investigation is first under new Digital Services Act (DSA)"

Why Gemini 2.5 Pro?

Excels at processing large amounts of context (15+ archive stories) and synthesizing them into coherent, well-structured narratives. Superior multi-document understanding and long-form content generation.
🔍 Archive Search
Technology: Semantic search on Reuters Text Archive
Integration: Works with all skills (triggered by orchestration layer)
Capability: Context-aware query generation
🔀 Orchestration Integration
The orchestration layer intelligently determines when archive search is needed:

Automatic Triggers:
• Update skill - Automatically searches for related stories when updating
• Blank Slate skill - Searches for New Content + Background Sources
• Generic queries - Triggered when user asks about past events or context

Decision Logic:
Orchestrator analyzes user intent and determines if archive context would enhance the response. If yes, archive search is triggered before skill execution.
Complete Search Process
1. Orchestration Decision: Orchestrator determines archive search is needed

2. Query Generation:
• Skill's respective AI model analyzes the request
• Generates targeted semantic search queries
• Creates multiple query variations for comprehensive results

3. Archive Retrieval:
• Semantic search executed on Reuters Text Archive
• Returns ranked results based on relevance
• Categorizes as New Content or Background Sources (Blank Slate)

4. User Selection:
• Relevant stories presented to user
• User can select which stories to include
• Options: Load more, Skip, Use Selected

5. Synthesis:
• Selected stories passed to AI model
• Model synthesizes archive content with new information
• Generates output following skill-specific structure
